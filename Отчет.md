# Отчёт — Этап 1: Мобильная адаптация

**Дата:** 28 марта 2026

## Что сделано

### 1. Анимации при скролле (уже было, проверено)
- IntersectionObserver для fade-in + slide-up всех секций на мобиле
- Stagger-задержки для карточек (0.1s), скилл-иконок (0.05s), timeline-элементов
- CSS-классы `mob-fade` / `mob-visible` через `transition: opacity + transform`

### 2. Touch-friendly карточки проектов
- `:active` эффект `scale(0.98)` + `border-color: accent` на project-card и project-category-card
- Горизонтальный скролл категорий на главной с `scroll-snap-type: x mandatory`
- Каждая карточка категории — `scroll-snap-align: center`, ширина 75vw
- Скрытый скроллбар для чистого вида

### 3. Hero-секция на мобиле
- Three.js и particles-canvas скрыты через `display:none!important`
- CSS-градиент с анимацией `gradientShift` (15s цикл)
- 12 плавающих CSS-звёзд (hero-star)
- Полупрозрачное фото кода (hero-code.jpg) как фоновый слой через `::after`
- Typewriter-эффект набора текста (было статичное, теперь анимированное)
- Каскадные fade-in для greeting → title → subtitle → CTA

### 4. Бургер-меню с каскадной анимацией
- При открытии: пункты появляются один за другим (stagger 70ms)
- Каждый пункт: `opacity:0 + translateY(20px)` → `opacity:1 + translateY(0)`
- При закрытии: мгновенное исчезновение (transition-delay: 0s)

### 5. Timeline (секция Опыт)
- Линия рисуется при скролле сверху вниз (CSS custom property `--tl-progress`)
- Привязана к scroll через `requestAnimationFrame` + getBoundingClientRect
- Точки (timeline-dot) появляются с scale-анимацией + glow при входе во viewport
- IntersectionObserver для каждого timeline-item отдельно

### 6. Микроанимации
- Ripple-эффект на кнопках (hero-cta, telegram-btn, back-btn) при touchstart
- Scale-эффект на кнопках при :active (0.95)
- Scale + background на project-tag при тапе
- Skill-item: scale(1.2) при :active

### 7. Фото через Pexels API
- Скачаны 10 тематических фото (medium размер, ~12-27KB каждое)
- `hero-code.jpg` — код на экране, для Hero-секции (полупрозрачный фон)
- Тематические фото для каждой категории (cat-frontend.jpg, cat-python.jpg, и т.д.)
- Фоновые изображения в карточках категорий через `.cat-bg` (opacity 0.12, hover 0.2)
- Сохранены локально в `assets/images/`

### 8. Производительность
- Кастомный курсор скрыт на мобильных (`@media (hover:none)`)
- Three.js, particles, GSAP, tilt, magnetic buttons — отключены на мобиле
- Hover-эффекты отключены (neonPulse, card-bg scale, etc.)
- `overflow-x: hidden` на html и body
- Все мобильные анимации используют только `transform` и `opacity` (GPU)

## Что использовано

- **IntersectionObserver** — анимации при скролле, timeline items
- **CSS transitions/animations** — fade-in, scale, ripple, gradient shift, stagger
- **CSS scroll-snap** — горизонтальный скролл категорий
- **requestAnimationFrame** — scroll-driven timeline линия
- **Pexels API** — тематические фотографии
- **CSS custom properties** — `--tl-progress`, `--stagger-delay`
- **Без тяжёлых библиотек** — всё на чистом CSS/JS

## Известные ограничения

- Фото из Pexels имеют размер medium (~12-27KB) — оптимально для мобиле
- Горизонтальный скролл категорий работает только на мобиле (< 768px), на десктопе — grid 3 колонки
- Все десктопные эффекты (Three.js, GSAP, particles, cursor) не тронуты
